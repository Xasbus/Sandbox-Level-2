JavaScript Keyword: class

          • A JavaScript class is like a template.
          • It predefines an object without creating it.
          • Usually created at the top of the JavaScript file.
          • Class syntax
              1. Keyword class
              2. Class name
                - Usually in Pascal Case.
              3. Curly braces
              4. Assign properties their values
                  - Use equal signs and semi-colons
          
            class MyCar {
              color = "Red";
              year = 2024;
          }
          
        
          -------------
          Pascal Case *Important Note*
          Each work starts with capitalized case. MyCar

          Class Vs. Object

          • In developer tools, a class has a name and length.
          • The JavaScript class is different from a JavaScript object.
          • The syntax of a class is different than an object.

Promise

          • A special class that runs a callback function.
          • The object indicates the status of the callback function.
          • Observations :
            - It's not asynchronous. It immediately runs my Function
            - Status shows pending even after myFunction has finished.
          • The status must be manually changed.
          • The callback function must indicate when it has finished.
          • The Promise sends a callback function (resolve();) to the promised function (myFunctionPromised).
          • The promised function (myFunctionPromised) of the callback function (resolve();) indicates that the Promise is fullfilled.
          • That callback function is known as the resolve function.
          • A Promise is meant to be used by asynchronous code.
          • The asynchronous code is supposed resolve to indicate when it's done.
          • 
          
            function myFunctionPromised(resolve) {
            display("Inside my myFunctionPomised. ");
            myParameter();
            }

            }
